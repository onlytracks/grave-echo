## Goal
Press `?` to open a modal help screen showing all available keybindings. Press `?` or
Esc to close. Same modal pattern as the inventory screen (GameState-based routing).

## What Exists
- `src/input/input-handler.ts` — `parseInput()`, InputEvent types
- `src/game.ts` — GameState enum, input routing per state
- Inventory screen already established the modal overlay pattern (GameState.Inventory)

## Keybindings to Display

```
╔════════════ Help ════════════╗
║                               ║
║  Movement                     ║
║    Arrow Keys   Move          ║
║    .            Pass turn     ║
║                               ║
║  Combat                       ║
║    Tab          Cycle targets ║
║    Space        Attack target ║
║                               ║
║  Items                        ║
║    e            Pick up item  ║
║    i            Inventory     ║
║    s            Swap weapon   ║
║    u            Use consumable║
║                               ║
║  Other                        ║
║    F1           Toggle debug  ║
║    ?            This help     ║
║    Esc / q      Quit          ║
║                               ║
║         [?/Esc] close         ║
╚═══════════════════════════════╝
```

Keep the list in a data structure so it stays maintainable as keybinds change.

## Requirements

### 1. Input
Add `"help"` to InputEvent. Map `?` key (0x3F) in `parseInput()`.

### 2. GameState
Add `Help` to GameState enum. `?` while Running → Help. `?` or Esc while Help → Running.

### 3. Rendering
Render as overlay on the game grid region (same approach as inventory screen).
Static content — no cursor, no interaction beyond closing.

### 4. Keybind Data
Store bindings as an array of `{ key: string; action: string; category: string }` so
rendering is data-driven and easy to update.

## Files to Change
- `src/input/input-handler.ts` — add `?` mapping
- `src/game.ts` — add GameState.Help, input routing, render call
- New file: `src/renderer/panels/help-screen.ts` — render function

## Definition of Done
- `?` opens help overlay listing all keybindings
- `?` or Esc closes it
- Game does not advance turns while help is open
- `bun test` passes
